## 🎯 核心功能对比

### ✨ 新增功能

#### 1. Live2D 交互增强
- **双指手势操作**：支持拖动和缩放 Live2D 模型
  - 实现文件：`services/Live2DService.ts`, `hooks/useLive2D.ts`
  - 提交：b1d3e12 (feat: 实现 Live2D 双指手势操作（拖动+缩放）)
  - 需要 GestureHandlerRootView 包裹应用根节点支持

#### 2. 语音打断功能
- **用户说话打断 AI 回复**：AI 语音播放期间，用户开口说话可立即停止 AI 播放
  - 核心设计：服务端检测用户活动 → 发送 `user_activity` → 客户端停止播放
  - 实现文档：[docs/specs/voice-interrupt.md](./specs/voice-interrupt.md)
  - 提交：3862c69 (feat: 实现语音打断功能)

#### 3. 角色切换系统
- **动态切换 Live2D 模型**：切换角色时自动切换对应的 Live2D 模型
- **事件驱动架构**：统一切换事件驱动，解耦角色切换逻辑
- **音频清空**：角色切换时自动清空音频队列
- **防竞态保护**：语音/文本模式切换时防止竞态条件
  - 实现文件：`services/Live2DService.ts`, `services/api/characters.ts`
  - 提交：457e828 (feat: 实现角色切换联动 Live2D 模型切换，统一切换事件驱动架构)

#### 4. 语音模式准备状态提示
- **脉冲动画**：语音准备阶段的视觉反馈
- **就绪提示**：明确告知用户系统已准备好接收语音输入
  - 提交：2448d28 (feat: 语音模式添加准备状态提示（脉冲动画 + 就绪反馈）)

#### 5. FRP 反向代理集成
- **开发环境支持**：支持通过 FRP 进行内网穿透，方便真机调试
  - 文档：[docs/guides/frp-integration.md](./guides/frp-integration.md) (假设存在)
  - 提交：fbdf8db (docs: 添加 FRP 反向代理集成文档)

#### 6. 音频格式协商
- **RN 端音频格式协商**：支持自定义语音播放格式
- **WebSocket 二进制接收修复**：修复了 RN 端 WebSocket 接收二进制数据的问题
  - 实现文件：`packages/project-neko-audio-service/src/native/audioServiceNative.ts`
  - 提交：632440f (feat: RN 端音频格式协商，支持自定义语音播放)

#### 7. 开发连接配置优化
- **环境变量配置**：支持通过 `.env` 配置开发连接参数
- **DevConnectionStorage**：本地存储开发连接配置
  - 实现文件：`utils/devConnectionConfig.ts`, `services/DevConnectionStorage.ts`
  - 提交：1eaa83c (chore: 支持环境变量配置开发连接参数)

## 🔄 从 v1.0.0-stable 的升级指南

### 必须操作
1. **更新 submodule**
   ```bash
   git submodule update --init --recursive
   ```

2. **重新安装依赖**
   ```bash
   npm install
   ```

3. **重新生成原生工程**（如有原生代码变更）
   ```bash
   npx expo prebuild --platform android --clean
   ```

### 可选配置
- 配置 `.env` 文件设置开发连接参数（参考 `.env.example`）
- 如需内网穿透，参考 FRP 文档配置
